teefaa bootstrap Beta

teefaa bootstrap is a set of documented shell scripts which helps managing system images among multiple Linux Distros, and also among multiple versions of system snapshots. The mission is to provide the scripts and methods to build a different type of cluster which provides the way for cluster users to run their own unique os images on bare-metal machines. The beta will be implemented in FutureGrid soon.

Quick Try Clone hostA onto hostB

  1. Prepare two Linux machines or VMs(call hostA and hostB). And reboot hostB with a LiveCD.

  2. Switch to root and create ssh-key pair on hostB and register the key on hostA's authorized_keys, so that you can login without password from hostB to hostA

    (hostB)$ sudo su - 
    (hostB)# ssh-keygen 
    (hostB)# key=$(cat .ssh/id_dsa.pub) 
    (hostB)# ssh hostA "echo $key >> .ssh/authorized_keys"

  3. Start the installation

    Before you start, you need to know this will overwrite /dev/sda and install the system. The /dev/sda is usually the first disk on a machine. If you want to check yours and need to change it, run "fdisk -l" then modify the DISK=sda on the localrc file.

    $ git@github.com:kjtanaka/teefaa.git 
    # cd teefaa; ./bootstrap.sh

    It takes a few minutes, we recommend reading the documentation while it is building.
